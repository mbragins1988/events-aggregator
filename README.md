# Events Aggregator

Сервис-агрегатор для работы с системой управления событиями и мероприятиями. Добавляет кэширование, фильтрацию, пагинацию, инкрементальную синхронизацию, гарантированную доставку уведомлений и идемпотентность операций.

## Технологический стек

- **Python 3.10+**
- **FastAPI** — веб-фреймворк
- **SQLAlchemy 2.0+** (async) — ORM
- **PostgreSQL** — база данных
- **Alembic** — миграции
- **httpx** — HTTP клиент
- **pytest** — тестирование
- **ruff** — линтер и форматтер
- **uv** — менеджер зависимостей
- **Docker** — контейнеризация
- **GitHub Actions** — CI/CD

## Функциональность

### API эндпоинты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/health` | Проверка доступности сервиса |
| GET | `/api/events` | Список событий с пагинацией и фильтрацией по дате |
| GET | `/api/events/{event_id}` | Детальная информация о событии |
| GET | `/api/events/{event_id}/seats` | Список свободных мест (кэширование 30 сек) |
| POST | `/api/tickets` | Регистрация на событие (с поддержкой идемпотентности) |
| DELETE | `/api/tickets/{ticket_id}` | Отмена регистрации |
| POST | `/api/sync/trigger` | Ручной запуск синхронизации |

### Ключевые особенности

#### Инкрементальная синхронизация
- Хранит метаданные синхронизации в таблице `sync_metadata`
- Загружает только новые или измененные события

### Фоновый воркер
- Запускается автоматически с приложением в main.py outbox_worker.start()
- Каждые 10 секунд проверяет таблицу outbox на наличие записей со статусом pending
- Для каждой записи формирует уведомление: "Вы успешно зарегистрированы на мероприятие - {event_name}"
- Отправляет запрос в Capashino API с idempotency_key = outbox_event_id
- При успехе (HTTP 201) помечает запись как sent
- При ошибке оставляет статус pending (повторная попытка на следующей итерации)

#### Transactional Outbox
- Гарантированная доставка уведомлений в Capashino
- Запись о событии сохраняется в той же транзакции, что и билет
- Отдельный воркер обрабатывает неотправленные записи каждые 10 секунд
- При ошибке повторяет попытки до успеха

#### Идемпотентность регистрации
- Поддержка `idempotency_key` в запросе `POST /api/tickets`
- Повторный запрос с тем же ключом возвращает существующий билет
- Конфликт данных (тот же ключ, другие данные) → 409 Conflict

#### Кэширование
- Список свободных мест кэшируется на 30 секунд
- Уменьшает нагрузку на внешний API

#### Мониторинг ошибок (GlitchTip)
- Интеграция через `sentry-sdk`
- DSN передается через переменную окружения `SENTRY_DSN`

## Установка и запуск

### Локальная разработка

```bash
# Клонирование репозитория
git clone https://github.com/your-username/events-aggregator.git
cd events-aggregator

# Создание виртуального окружения
uv venv
source .venv/bin/activate  # или .venv\Scripts\activate на Windows

# Установка зависимостей
uv sync

# Настройка переменных окружения
cp .env.example .env
# Отредактируйте .env, добавьте POSTGRES_CONNECTION_STRING, API_TOKEN и т.д.

# Применение миграций
alembic upgrade head

# Запуск приложения
uvicorn app.main:app --reload
